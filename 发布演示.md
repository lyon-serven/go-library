# MyLib 发布演示 - 完整流程

## 📋 问题总结

您遇到的问题：
1. ❌ 即使 Gitee 仓库是公开的，Go 默认也可能无法正确解析和下载
2. ❌ 仓库可能不存在或者没有正确配置 Go module

## ✅ 解决方案

### 核心问题

当前的 `go.mod` 配置：
```go
module mylib  // ❌ 错误：本地模块名，其他项目无法引用
```

需要改为：
```go
module github.com/yourusername/mylib  // ✅ 正确：完整的仓库路径
```

## 🎯 完整解决流程

### 场景1: 使用 GitHub（推荐 ⭐）

#### 第1步: 创建 GitHub 仓库

1. 登录 GitHub: https://github.com
2. 点击 "New repository"
3. 仓库名: `mylib`
4. 设置为 Public（公开）
5. 不要勾选 "Initialize with README"（我们已经有代码）
6. 点击 "Create repository"

#### 第2步: 运行自动化脚本

假设您的 GitHub 用户名是 `yourname`：

```powershell
# 在 PowerShell 中运行
cd e:\2.Source\Go\src\mylib

# 先预览（不修改文件）
.\update-module-path.ps1 -RepoURL "github.com/yourname/mylib" -DryRun

# 确认无误后，执行实际更新
.\update-module-path.ps1 -RepoURL "github.com/yourname/mylib"

# 按 'y' 确认
```

脚本会自动完成：
- ✅ 更新 `go.mod` 中的 module 名称
- ✅ 替换所有 `.go` 文件中的导入路径
- ✅ 运行 `go mod tidy`
- ✅ 测试编译

#### 第3步: 推送到 GitHub

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "feat: initial commit - complete Go utility library with Google Authenticator"

# 连接到 GitHub 仓库（替换为您的仓库地址）
git remote add origin https://github.com/yourname/mylib.git

# 推送到 main 分支
git branch -M main
git push -u origin main

# 创建版本标签（重要！）
git tag -a v1.0.0 -m "Release v1.0.0 - Full featured utility library"
git push origin v1.0.0
```

#### 第4步: 在其他项目中使用

```bash
# 创建新项目测试
mkdir test-project
cd test-project
go mod init test-project

# 下载依赖
go get github.com/yourname/mylib@v1.0.0

# 创建测试代码
cat > main.go << 'EOF'
package main

import (
    "fmt"
    "github.com/yourname/mylib/util/authutil"
)

func main() {
    // 使用 Google Authenticator 功能
    secret, qrURL, err := authutil.QuickGenerate("MyApp", "user@example.com")
    if err != nil {
        panic(err)
    }
    
    fmt.Printf("密钥: %s\n", secret)
    fmt.Printf("二维码: %s\n", qrURL)
    
    // 生成验证码
    code, _ := authutil.QuickGenerateCode(secret)
    fmt.Printf("当前验证码: %s\n", code)
}
EOF

# 运行
go run main.go
```

**期望输出:**
```
密钥: ABCDEFGH12345678...
二维码: https://api.qrserver.com/v1/create-qr-code/...
当前验证码: 123456
```

### 场景2: 使用 Gitee（国内用户）

#### 第1步: 创建 Gitee 仓库

1. 登录 Gitee: https://gitee.com
2. 点击 "+" -> "新建仓库"
3. 仓库名: `mylib`
4. 设置为 **公开**
5. 取消勾选 "使用Readme文件初始化这个仓库"
6. 点击 "创建"

#### 第2步: 运行自动化脚本

假设您的 Gitee 用户名是 `yourname`：

```powershell
cd e:\2.Source\Go\src\mylib
.\update-module-path.ps1 -RepoURL "gitee.com/yourname/mylib"
```

#### 第3步: 推送到 Gitee

```bash
git init
git add .
git commit -m "feat: initial commit"
git remote add origin https://gitee.com/yourname/mylib.git
git branch -M master  # Gitee 默认用 master
git push -u origin master
git tag v1.0.0
git push origin v1.0.0
```

#### 第4步: 配置 Go 环境支持 Gitee

**重要！** Gitee 需要额外配置：

```bash
# 配置 Go 代理（使用国内镜像）
go env -w GOPROXY=https://goproxy.cn,direct

# 配置私有仓库路径（即使是公开仓库也建议配置）
go env -w GOPRIVATE=gitee.com/yourname/*
```

#### 第5步: 在其他项目中使用

```bash
# 在使用项目中也要配置环境
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourname/*

# 下载依赖
go get gitee.com/yourname/mylib@v1.0.0
```

**如果还是有问题，使用 replace 方案:**

在使用项目的 `go.mod` 中添加：
```go
module your-project

go 1.23

require gitee.com/yourname/mylib v1.0.0

// 临时解决方案：使用 replace
replace gitee.com/yourname/mylib => github.com/yourname/mylib v1.0.0
```

## 🐛 常见错误及解决

### 错误1: "module not found"

```
go: downloading github.com/yourname/mylib v1.0.0
go: github.com/yourname/mylib@v1.0.0: reading https://proxy.golang.org/github.com/yourname/mylib/@v/v1.0.0.info: 404 Not Found
```

**原因**: 
- 仓库不存在
- 没有推送代码
- 没有创建版本标签

**解决**:
```bash
# 确认已推送
git push origin main

# 确认标签已推送
git push origin v1.0.0

# 等待 5-10 分钟让 Go 代理同步
# 然后重试
go get github.com/yourname/mylib@v1.0.0
```

### 错误2: "invalid version: module contains a go.mod file"

**原因**: go.mod 中的 module 名称与仓库路径不匹配

**解决**:
```bash
# 检查 go.mod 第一行
cat go.mod | head -n 1

# 应该是: module github.com/yourname/mylib
# 如果不是，修改后重新提交
git add go.mod
git commit -m "fix: correct module path"
git push

# 删除旧标签，创建新标签
git tag -d v1.0.0
git push origin :refs/tags/v1.0.0
git tag v1.0.0
git push origin v1.0.0
```

### 错误3: Gitee 无法下载

```
go: gitee.com/yourname/mylib@v1.0.0: invalid version: git ls-remote failed
```

**解决方案A: 配置代理**
```bash
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourname/*
go clean -modcache
go get gitee.com/yourname/mylib@v1.0.0
```

**解决方案B: 使用 replace（推荐）**

在使用项目的 `go.mod` 中：
```go
require gitee.com/yourname/mylib v1.0.0

replace gitee.com/yourname/mylib => github.com/yourname/mylib-mirror v1.0.0
```

**解决方案C: 同时发布到 GitHub 和 Gitee**
```bash
# 添加两个远程仓库
git remote add github https://github.com/yourname/mylib.git
git remote add gitee https://gitee.com/yourname/mylib.git

# 推送到两个仓库
git push github main
git push gitee master

# 推送标签
git push github v1.0.0
git push gitee v1.0.0
```

## 📝 实际操作示例

### 完整的命令序列（GitHub）

```bash
# 假设您的 GitHub 用户名是: myusername
# 仓库已创建: https://github.com/myusername/mylib

cd e:\2.Source\Go\src\mylib

# 1. 运行配置脚本
.\update-module-path.ps1 -RepoURL "github.com/myusername/mylib"
# 提示时输入 'y'

# 2. 初始化 Git
git init

# 3. 添加文件
git add .

# 4. 提交
git commit -m "feat: initial commit - Go utility library v1.0.0"

# 5. 连接远程仓库
git remote add origin https://github.com/myusername/mylib.git

# 6. 推送
git branch -M main
git push -u origin main

# 7. 创建标签
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0

# 8. 验证（在新目录测试）
cd ..
mkdir test-mylib
cd test-mylib
go mod init test
go get github.com/myusername/mylib@v1.0.0

# 9. 创建测试文件
echo 'package main
import ("fmt"; "github.com/myusername/mylib/util/authutil")
func main() {
    s, q, _ := authutil.QuickGenerate("App", "user@example.com")
    fmt.Println("Secret:", s[:20]+"...", "\nQR:", q)
}' > main.go

# 10. 运行测试
go run main.go
```
