# 如何使用更新脚本

## 🚀 快速使用

### 方法1: 使用新的简化脚本（推荐）

```powershell
# 预览模式（不修改文件，先看看会改什么）
.\update-module.ps1 -RepoURL "github.com/yourname/mylib" -DryRun

# 实际执行
.\update-module.ps1 -RepoURL "github.com/yourname/mylib"
```

### 方法2: 手动替换（如果脚本有问题）

如果脚本无法运行，可以手动操作：

#### 步骤1: 修改 go.mod

打开 `go.mod` 文件，将第一行改为：

```go
module github.com/yourname/mylib
```

#### 步骤2: 批量替换导入路径

在 VS Code 中：
1. 按 `Ctrl + Shift + H` 打开全局搜索替换
2. 搜索: `"mylib/`
3. 替换为: `"github.com/yourname/mylib/`
4. 点击"全部替换"

或使用 PowerShell：

```powershell
# 简单的批量替换
Get-ChildItem -Path . -Filter *.go -Recurse | ForEach-Object {
    $content = Get-Content $_.FullName -Raw
    if ($content -match '"mylib/') {
        $content = $content -replace '"mylib/', '"github.com/yourname/mylib/'
        $content | Set-Content $_.FullName -NoNewline
        Write-Host "Updated: $($_.Name)"
    }
}
```

#### 步骤3: 测试

```bash
go mod tidy
go build ./...
```

## 📋 完整示例

假设您的 GitHub 用户名是 `zhangsan`，仓库名是 `mylib`：

### 使用脚本：

```powershell
cd e:\2.Source\Go\src\mylib

# 先预览
.\update-module.ps1 -RepoURL "github.com/zhangsan/mylib" -DryRun

# 确认无误后执行
.\update-module.ps1 -RepoURL "github.com/zhangsan/mylib"
# 提示时输入 'y' 确认
```

### 然后推送到 GitHub：

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交

# 连接到 GitHub
git remote add origin https://github.com/zhangsan/mylib.git

# 推送
git branch -M main
git push -u origin main

# 创建版本标签
git tag v1.0.0
git push origin v1.0.0
```


## 🛠️ 脚本参数说明

### -RepoURL（必需）

仓库的完整路径，格式：`平台/用户名/仓库名`

支持的平台：
- `github.com` - GitHub（推荐）
- `gitee.com` - Gitee（国内）

示例：
```powershell
-RepoURL "github.com/yourname/mylib"
-RepoURL "gitee.com/yourname/mylib"
```

### -DryRun（可选）

预览模式，不实际修改文件，只显示会做什么改动。

```powershell
# 预览
.\update-module.ps1 -RepoURL "github.com/yourname/mylib" -DryRun

# 实际执行
.\update-module.ps1 -RepoURL "github.com/yourname/mylib"
```

## ⚠️ 常见问题

### Q1: 脚本报错 "无法运行脚本"

**问题**: PowerShell 执行策略限制

**解决**:
```powershell
# 临时允许执行脚本（当前会话）
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 然后运行脚本
.\update-module.ps1 -RepoURL "github.com/yourname/mylib"
```

### Q2: 脚本报错 "字符编码问题"

**解决**: 使用手动方法（见上文"方法2"）

### Q3: 不知道自己的 GitHub 用户名

登录 GitHub，查看右上角头像，或访问 https://github.com/settings/profile

## 📝 检查清单

运行脚本后，检查这些项目：

```bash
# 1. 检查 go.mod 第一行
head -n 1 go.mod

# 2. 检查导入路径（随机检查一个文件）
grep "import" integration_demo.go | head -n 5
# 应该看到 github.com/yourname/mylib 而不是 mylib

# 3. 测试编译
go mod tidy
go build ./...
# 应该没有错误

# 4. 运行测试
go run integration_demo.go
# 应该正常运行
```

## 💡 提示

1. **建议先用 -DryRun 预览**: 可以看到会改什么，不用担心弄坏代码

2. **提交前备份**: 运行脚本前先提交当前代码或创建备份分支

3. **优先使用 GitHub**: 比 Gitee 配置更简单，Go 工具链原生支持

4. **记得创建标签**: 没有版本标签，其他项目无法下载
