# MyLib Go 模块配置指南

## 问题说明

当前项目的 `go.mod` 配置为本地模块名 `mylib`，无法被其他项目通过网络引用。需要配置为正确的远程仓库路径。

## 解决方案

### 方案1: 使用 GitHub（推荐）

#### 1. 创建 GitHub 仓库

1. 登录 GitHub，创建新仓库，例如: `https://github.com/yourusername/mylib`
2. 可以设置为 public（公开）或 private（私有）

#### 2. 修改 go.mod

将 `go.mod` 的第一行修改为：

```go
module github.com/yourusername/mylib
```

完整示例：
```go
module github.com/yourusername/mylib

go 1.23

require (
	github.com/fsnotify/fsnotify v1.9.0
	golang.org/x/sys v0.13.0
	gopkg.in/yaml.v3 v3.0.1
)
```

#### 3. 更新所有代码中的导入路径

需要全局替换所有导入语句：

**之前:**
```go
import "mylib/util/authutil"
import "mylib/cache"
import "mylib/config"
```

**之后:**
```go
import "github.com/yourusername/mylib/util/authutil"
import "github.com/yourusername/mylib/cache"
import "github.com/yourusername/mylib/config"
```

#### 4. 推送到 GitHub

```bash
cd e:\2.Source\Go\src\mylib
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/mylib.git
git push -u origin main
```

#### 5. 创建版本标签（重要！）

```bash
git tag v1.0.0
git push origin v1.0.0
```

#### 6. 其他项目使用

```bash
# 在其他项目中使用
go get github.com/yourusername/mylib@v1.0.0

# 或者最新版本
go get github.com/yourusername/mylib@latest
```

```go
// 在代码中使用
import "github.com/yourusername/mylib/util/authutil"
```

### 方案2: 使用 Gitee（国内用户）

#### 1. 创建 Gitee 仓库

1. 登录 Gitee，创建新仓库，例如: `https://gitee.com/yourusername/mylib`
2. 设置为公开仓库

#### 2. 修改 go.mod

```go
module gitee.com/yourusername/mylib

go 1.23

require (
	github.com/fsnotify/fsnotify v1.9.0
	golang.org/x/sys v0.13.0
	gopkg.in/yaml.v3 v3.0.1
)
```

#### 3. 配置 Go 环境支持 Gitee

由于 Go 默认支持 GitHub，使用 Gitee 需要额外配置：

**方法A: 使用 GOPROXY（推荐）**

```bash
# 使用国内代理
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourusername/*
```

**方法B: 配置 Git 凭据**

```bash
# 配置 Git 使用 HTTPS
git config --global url."https://gitee.com/".insteadOf "https://gitee.com/"
```

#### 4. 推送到 Gitee

```bash
cd e:\2.Source\Go\src\mylib
git init
git add .
git commit -m "Initial commit"
git branch -M master
git remote add origin https://gitee.com/yourusername/mylib.git
git push -u origin master
```

#### 5. 创建版本标签

```bash
git tag v1.0.0
git push origin v1.0.0
```

#### 6. 其他项目使用（Gitee 特殊配置）

**使用项目配置:**

```bash
# 在使用 mylib 的项目中配置
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourusername/*

# 下载依赖
go get gitee.com/yourusername/mylib@v1.0.0
```

**或者使用 replace 指令（开发阶段）:**

在使用项目的 `go.mod` 中：
```go
module yourproject

go 1.23

require gitee.com/yourusername/mylib v1.0.0

// 如果直接下载有问题，可以用 replace
replace gitee.com/yourusername/mylib => ./path/to/local/mylib
```

### 方案3: 私有仓库配置

如果使用私有仓库，需要额外配置：

#### GitHub 私有仓库

```bash
# 配置 Git 凭据
git config --global url."https://${GITHUB_TOKEN}@github.com/".insteadOf "https://github.com/"

# 或使用 SSH
git config --global url."git@github.com:".insteadOf "https://github.com/"

# 配置 Go
go env -w GOPRIVATE=github.com/yourusername/*
```

#### Gitee 私有仓库

```bash
# 配置访问令牌
git config --global url."https://oauth2:${GITEE_TOKEN}@gitee.com/".insteadOf "https://gitee.com/"

# 配置 Go
go env -w GOPRIVATE=gitee.com/yourusername/*
```

## 完整迁移步骤（以 GitHub 为例）

### 第1步: 修改 go.mod

```bash
cd e:\2.Source\Go\src\mylib
```

编辑 `go.mod`:
```go
module github.com/yourusername/mylib

go 1.23

require (
	github.com/fsnotify/fsnotify v1.9.0
	golang.org/x/sys v0.13.0
	gopkg.in/yaml.v3 v3.0.1
)
```

### 第2步: 批量替换导入路径

使用 PowerShell 批量替换：

```powershell
# 查找所有 .go 文件中的导入
Get-ChildItem -Path . -Filter *.go -Recurse | ForEach-Object {
    (Get-Content $_.FullName) -replace 'import "mylib/', 'import "github.com/yourusername/mylib/' | Set-Content $_.FullName
}

# 或使用更精确的替换
Get-ChildItem -Path . -Filter *.go -Recurse | ForEach-Object {
    $content = Get-Content $_.FullName -Raw
    $content = $content -replace '"mylib/cache"', '"github.com/yourusername/mylib/cache"'
    $content = $content -replace '"mylib/config"', '"github.com/yourusername/mylib/config"'
    $content = $content -replace '"mylib/util/authutil"', '"github.com/yourusername/mylib/util/authutil"'
    $content = $content -replace '"mylib/util/cryptoutil"', '"github.com/yourusername/mylib/util/cryptoutil"'
    $content = $content -replace '"mylib/util/httputil"', '"github.com/yourusername/mylib/util/httputil"'
    $content = $content -replace '"mylib/util/timeutil"', '"github.com/yourusername/mylib/util/timeutil"'
    $content = $content -replace '"mylib/stringutil"', '"github.com/yourusername/mylib/stringutil"'
    $content = $content -replace '"mylib/sliceutil"', '"github.com/yourusername/mylib/sliceutil"'
    $content = $content -replace '"mylib/fileutil"', '"github.com/yourusername/mylib/fileutil"'
    $content = $content -replace '"mylib/validator"', '"github.com/yourusername/mylib/validator"'
    $content | Set-Content $_.FullName -NoNewline
}
```

## 常见问题解决

### Q1: go get 报错 "module not found"

**原因**: 仓库可能没有正确的 go.mod 文件或版本标签

**解决**:
```bash
# 确保仓库根目录有 go.mod
# 确保推送了版本标签
git tag v1.0.0
git push origin v1.0.0

# 如果是 main 分支，确保 GitHub 默认分支是 main
```

### Q2: Gitee 无法下载

**原因**: Go 默认不支持 Gitee

**解决**:
```bash
# 配置代理
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourusername/*

# 如果还不行，使用 replace
# 在使用项目的 go.mod 中添加:
replace gitee.com/yourusername/mylib => github.com/yourusername/mylib v1.0.0
```

### Q3: 导入路径还是旧的

**原因**: 没有更新所有文件的导入路径

**解决**:
```bash
# 全局搜索并替换
grep -r "import \"mylib/" . --include="*.go"
# 手动或使用脚本批量替换
```

### Q4: go mod tidy 报错

**原因**: 循环依赖或路径错误

**解决**:
```bash
# 清理缓存
go clean -modcache
rm go.sum

# 重新整理
go mod tidy
```

## 推荐的版本管理策略

### 语义化版本（Semantic Versioning）

- `v1.0.0` - 首个稳定版本
- `v1.0.1` - Bug 修复
- `v1.1.0` - 新增功能（向后兼容）
- `v2.0.0` - 重大变更（不向后兼容）

### 标签命名规范

```bash
# 主要版本
git tag v1.0.0
git tag v2.0.0

# 预发布版本
git tag v1.0.0-beta.1
git tag v1.0.0-rc.1

# 推送标签
git push origin v1.0.0
```

### 使用特定版本

```bash
# 使用最新版本
go get github.com/yourusername/mylib@latest

# 使用特定版本
go get github.com/yourusername/mylib@v1.0.0

# 使用特定 commit
go get github.com/yourusername/mylib@commit-hash
```
