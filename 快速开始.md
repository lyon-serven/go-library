# 快速开始指南

## 📦 发布到仓库的完整流程

### 准备工作

确保您已经：
1. ✅ 在 GitHub 或 Gitee 上创建了仓库
2. ✅ 安装了 Git 和 Go 1.18+

---

## 🚀 方法1: 使用自动化脚本（推荐）

### 步骤1: 运行配置脚本

**GitHub:**
```powershell
.\update-module-path.ps1 -RepoURL "github.com/yourusername/mylib"
```

**Gitee:**
```powershell
.\update-module-path.ps1 -RepoURL "gitee.com/yourusername/mylib"
```

**预览模式（不实际修改）:**
```powershell
.\update-module-path.ps1 -RepoURL "github.com/yourusername/mylib" -DryRun
```

### 步骤2: 推送到仓库

脚本运行完成后，按照提示执行：

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "feat: initial commit with all features"

# 添加远程仓库
git remote add origin https://github.com/yourusername/mylib.git

# 推送
git branch -M main
git push -u origin main

# 创建版本标签
git tag v1.0.0
git push origin v1.0.0
```

### 步骤3: 在其他项目中使用

```bash
go get github.com/yourusername/mylib@v1.0.0
```

---

## 🔧 方法2: 手动配置

### 步骤1: 修改 go.mod

编辑 `go.mod` 第一行：

```go
module github.com/yourusername/mylib
```

### 步骤2: 全局替换导入路径

使用 PowerShell 批量替换：

```powershell
# 替换所有导入语句
Get-ChildItem -Path . -Filter *.go -Recurse | ForEach-Object {
    $content = Get-Content $_.FullName -Raw
    $content = $content -replace '"mylib/', '"github.com/yourusername/mylib/'
    $content | Set-Content $_.FullName -NoNewline
}
```

或使用 VS Code 的全局搜索替换功能：
- 搜索: `"mylib/`
- 替换为: `"github.com/yourusername/mylib/`

### 步骤3: 测试编译

```bash
go mod tidy
go build ./...
go test ./...
```

### 步骤4: 推送到仓库

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/mylib.git
git branch -M main
git push -u origin main
git tag v1.0.0
git push origin v1.0.0
```

## 🇨🇳 Gitee 特殊配置

如果使用 Gitee，需要额外配置：

### 配置 Go 环境

```bash
# 使用国内代理
go env -w GOPROXY=https://goproxy.cn,direct

# 配置私有仓库
go env -w GOPRIVATE=gitee.com/yourusername/*
```

### 推送到 Gitee

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://gitee.com/yourusername/mylib.git
git branch -M master  # Gitee 默认使用 master
git push -u origin master
git tag v1.0.0
git push origin v1.0.0
```

### 在其他项目中使用

```bash
# 配置环境
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourusername/*

# 下载依赖
go get gitee.com/yourusername/mylib@v1.0.0
```

## 📋 常见问题

### Q: go get 报错 "module not found"

**可能原因:**
1. 仓库不存在或设置为私有
2. 没有推送版本标签
3. go.mod 中的 module 名称不匹配

**解决方法:**
```bash
# 确保仓库是公开的
# 确保推送了代码和标签
git push origin main
git tag v1.0.0
git push origin v1.0.0

# 等待几分钟让 Go 代理同步
```

### Q: 导入路径错误

**确认事项:**
- ✅ go.mod 中的 module 名称正确
- ✅ 所有 .go 文件的 import 已更新
- ✅ 运行了 `go mod tidy`

### Q: Gitee 无法下载

**解决方法:**
```bash
# 配置代理和私有仓库
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=gitee.com/yourusername/*

# 清理缓存
go clean -modcache

# 重试
go get gitee.com/yourusername/mylib@v1.0.0
```

## 📚 版本管理

### 创建新版本

```bash
# 修改代码...
git add .
git commit -m "feat: add new feature"
git push

# 创建新版本标签
git tag v1.1.0
git push origin v1.1.0
```

### 版本号规范

- `v1.0.0` - 首次稳定版本
- `v1.0.1` - Bug 修复
- `v1.1.0` - 新增功能（向后兼容）
- `v2.0.0` - 重大变更（不兼容旧版本）

### 更新依赖

```bash
# 更新到最新版本
go get github.com/yourusername/mylib@latest

# 更新到特定版本
go get github.com/yourusername/mylib@v1.1.0

# 更新所有依赖
go get -u ./...
```
